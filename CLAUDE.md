```
   "나는 스타트업 마케팅 담당자고, 신제품 출시를 앞두고 있어"

2. 목표 (뭘 원하는지)
   "론칭 이벤트 기획안을 만들고 싶어"

3. 제약 조건 (한계/규칙)
   "예산은 500만원이고, 온라인 중심이어야 해"

4. 결과물 형식
   "PPT 10페이지로 경영진에게 보고할 거야"

5. 참고 자료 (있다면)
   "작년 이벤트 결과 보고서 첨부할게"

💡 이 5단계만 기억하면 90% 성공!
```

### 나쁜 예 vs 좋은 예

❌ **나쁜 예**:
```
"보고서 만들어줘"
```

✅ **좋은 예**:
```
"우리 팀의 3분기 매출 보고서를 만들어줘.
- 데이터: 첨부한 엑셀 파일 참고
- 포함 내용: 월별 매출, 전년 대비 증감률, Top 5 제품
- 형식: PPT 10페이지 이내
- 톤: 경영진 보고용 (전문적, 간결)
- 마감: 내일 오전까지

우리 회사 PPT 템플릿은 첨부 파일 확인해줘."
```

## 🤖 **멀티 에이전트 활용 사례** (4가지)

### 1️⃣ 병렬화 가능한 작업 (Map-Reduce)

```
예시: 연말 결산 보고서
- 1월 매출 분석 (서브 에이전트 1)
- 2월 매출 분석 (서브 에이전트 2)
- 3월 매출 분석 (서브 에이전트 3)
...
- 12월 매출 분석 (서브 에이전트 12)

→ 동시 진행 → 12배 빨라짐
```

### 2️⃣ 테스트 타임 컴퓨트

```
예시: 중요한 비즈니스 의사결정
"신규 시장 진출, A국 vs B국 어디가 좋을까?"

서브 에이전트 1: A국 분석 → 결론 도출
서브 에이전트 2: B국 분석 → 결론 도출
서브 에이전트 3: 리스크 중심 분석 → 결론 도출
서브 에이전트 4: 수익성 중심 분석 → 결론 도출
서브 에이전트 5: 문화적 차이 중심 분석 → 결론 도출

부모 에이전트: 5개 결론 비교 → 최선의 답 선택

Eric의 말:
"여러 사람이 머리 맞대면 더 좋은 결과 나오잖아요. Claude도 마찬가지입니다"
```

### 3️⃣ 다양한 접근법 시도

```
예시: 마케팅 카피 작성
"신제품 광고 문구 만들어줘"

서브 에이전트 1: 유머러스한 톤 (2030 타겟)
서브 에이전트 2: 전문적인 톤 (기업 고객 타겟)
서브 에이전트 3: 감성적인 톤 (4050 타겟)
서브 에이전트 4: 데이터 중심 (이성적 소비자 타겟)

→ 4개 결과 비교 → 마음에 드는 것 선택 or 조합
```

### 4️⃣ 도구 분할 관리 ⭐ 실무 핵심

```
문제: 대기업은 도구가 100~200개

❌ 기존 방식:
- Claude 1명에게 200개 도구 설명서 전부 줌
- Claude: "헷갈려... 뭘 써야 하지?"
- 잘못된 도구 선택 빈번
- 느리고 부정확

✅ 멀티 에이전트 방식:

부모 Claude: "음, 이건 회계 관련이네. 회계 서브 에이전트 호출!"
서브 에이전트 (회계): 20개 회계 도구만 가지고 있음
                      → 빠르고 정확하게 올바른 도구 선택

도구 분류 예시:
- 서브 에이전트 A: 회계 도구 20개
- 서브 에이전트 B: 마케팅 도구 20개
- 서브 에이전트 C: HR 도구 20개
- 서브 에이전트 D: 영업 도구 20개
- 서브 에이전트 E: 물류 도구 20개
```

## 🎯 **실전 프롬프트 템플릿** (바로 복사해서 쓰세요)

### 템플릿 1: 경쟁사 분석

```
"경쟁사 [회사명 리스트]를 분석해줘.

각 회사별로 다음 정보 수집:
1. 주요 제품/서비스
2. 가격 정책
3. 최근 1년 주요 뉴스
4. 강점/약점

결과는 비교 표 형식으로,
우리 회사([우리 회사명])와 비교해서
차별화 포인트 3가지 제시해줘.

형식: PPT 10페이지
톤: 임원 보고용"

💡 멀티 에이전트가 자동으로 각 회사 동시 분석!
```

### 템플릿 2: 마케팅 콘텐츠 대량 생성

```
"Instagram 피드 포스트 30개 만들어줘.

주제: [제품/서비스명]
타겟: [연령대, 성별, 관심사]
톤: [친근한/전문적인/유머러스한]

각 포스트마다 포함:
- 제목 (10자 이내)
- 본문 (100자 이내)
- 해시태그 10개
- 이미지 설명 (Canva에서 만들 수 있게)

형식: 엑셀 스프레드시트
(날짜별로 정리해서 바로 일정 등록 가능하게)"

💡 30개를 10개씩 3개 서브 에이전트가 동시 생성!
```

### 템플릿 3: 회의록 → 실행 계획

```
"첨부한 회의록을 분석해서 실행 계획을 만들어줘.

추출할 내용:
1. 결정 사항 리스트
2. 액션 아이템 (담당자, 마감일 포함)
3. 미결 사항 (다음 회의 안건)

그 다음:
4. 우선순위 매기기 (긴급도 × 중요도)
5. 타임라인 (간트 차트 형식)
6. 리스크 요인 3가지

결과물:
- 요약 보고서 (1페이지)
- 액션 아이템 트래커 (엑셀)
- 다음 회의 안건 (워드)"

💡 Claude가 회의록 읽고 → 구조화 → 3개 파일 생성
```

### 템플릿 4: 데이터 분석 → 인사이트

```
"첨부한 판매 데이터(엑셀)를 분석해서 인사이트를 뽑아줘.

데이터 포함 내용:
- 날짜, 제품, 수량, 금액, 지역, 고객 유형

분석 요청:
1. 월별 매출 추이 (차트)
2. 제품별 판매량 TOP 10 (파이 차트)
3. 지역별 성장률 (지도 시각화 설명)
4. 고객 유형별 구매 패턴

인사이트:
- 매출 증가/감소 원인 3가지
- 다음 분기 예측
- 액션 아이템 5가지

형식: PPT 15페이지 (경영진 보고용)"

💡 Claude가 데이터 읽고 → 분석 코드 작성 → 실행 → PPT 생성
```

### 템플릿 5: 이메일 자동 분류/답변

```
"고객 문의 이메일 [XX개 첨부]를 분류하고 답변 초안 만들어줘.

분류 기준:
1. 긴급 (24시간 내 답변 필요)
2. 일반 문의
3. 클레임
4. 제안/피드백

각 이메일마다:
- 카테고리 표시
- 핵심 내용 요약 (1줄)
- 답변 초안 (우리 회사 톤: 정중하고 전문적)
- 필요 시 에스컬레이션 여부

결과물:
엑셀 스프레드시트 (필터링 가능하게)
+ 긴급 케이스 별도 보고서"

💡 서브 에이전트들이 이메일 동시 처리 → 분류 → 답변 생성
```
